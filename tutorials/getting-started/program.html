First Program

This tutorial covers the installation of your IDE, creating the solution, and installing the library. If you have an IDE already setup and are familiar with project creation, skip to getting the NuGet package.

IDE

It's a must, for working with a library of this size, to have a good IDE on your hands. It'll simplify writing, debugging, and optimizing your code base.

We recommend:

  • Visual Studio 2019 or later:
    • Windows-only
    • has a free Community edition
  • Rider:
    • cross-platform
    • requires a license (free for open source)
Workload

Depending on the IDE you chose, the workload installation will differ slightly, but should be more or less the same. You will need to install the .NET desktop development workload which includes the latest .NET SDK.

Creating the Solution

Open Visual Studio. You should see the following screen appear by default:

Click Create a new project. In the following screen select Console Application. Make sure it's described as a .NET Core application.

Continue. In the following screen select Console Application. Make sure it's described as a .NET Core application.

Give the application a name. Just don't use something that'd collide in any way with the Disqord namespace to avoid issues down the line.

Select the framework to be .NET 5.0 or higher.

You should see Visual Studio open the solution, which is a barebones "hello world" application:

Getting the NuGet Package

It's time to grab the NuGet package, i.e. download the library so that we can use it in our code.

Stable Builds
Currently, only nightly builds are available.
Nightly Builds

Nightly builds are library builds published instantly after a commit is made to the GitHub repository, meaning that if you want to always be up-to-date you should prefer nightly builds.

These are available via the following MyGet package source: https://www.myget.org/F/disqord/api/v3/index.json.

To add this feed to your Visual Studio package source see Microsoft's tutorial.

Now to install the NuGet package, right click on your project in the solution explorer and choose Manage NuGet Packages.

On the NuGet screen, click on Browse, and choose the appropriate package source.

Check Include prerelease if you're using the MyGet package source.

Now install the following packages, in any order:

  • Disqord.Bot
  • Microsoft.Extensions.Hosting
  • Serilog.Extensions.Hosting
  • Serilog.Sinks.Console

Using the Library

Let's write a simple, but clean startup for our bot that can be easily expanded on later on: 

using Disqord.Bot.Hosting;
using Microsoft.Extensions.Hosting;
using Serilog;

namespace MyBot
{
    class Program
    {
        private static void Main(string[] args)
        {
            using (var host = CreateHost())
            {
                host.Run();
            }
        }

        private static IHost CreateHost()
        {
            // All host configuration goes here.
            return new HostBuilder()
                .UseSerilog((context, logger) =>
                {
                    // Instructs Serilog to log into the console.
                    logger.WriteTo.Console();
                })
                .ConfigureDiscordBot((context, bot) =>
                {
                    // Sets the bot token.
                    bot.Token = "YOUR_BOT_TOKEN";

                    // Allows ? and ! as prefixes for commands.
                    // You can also mention the bot.
                    bot.Prefixes = new[] { "?", "!" };
                })
                .Build();
        }
    }
}

Creating a Command Module

To create commands, right click on your project in the solution explorer and choose Add > Class

Give the module a name. I called it BotModule, because I intend on only having bot-related commands in it. The name can be anything and you can have as many modules as you want.

Now, in our new class write the following code in: 

using Disqord.Bot;
using Qmmands;

namespace MyBot
{
    public class BotModule : DiscordModuleBase
    {
        [Command("ping")]
        public DiscordCommandResult Ping()
        {
            return Response("Pong!");
        }
    }
}

This adds a basic ping-pong command. Let's try it out! Boot up your bot and type ?ping, !ping or @YourBot ping in a channel.
